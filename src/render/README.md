# src › render

Rendering pipeline and ray-tracing engine.

- `integrators/` – Dedicated modules for the renderer’s sampling strategies.
  - `integrator_common.c` – Owns the `IntegratorContext`, tile allocator/clear helpers, RNG seeding, and the `IrradianceCache` container used by the camera pass. Any integrator that opts into tile-based accumulation or cache population goes through these helpers so lifetime/clearing stays consistent.
  - `forward_light_integrator.c` – Emits BSDF-driven light paths using the same Lambert/GGX materials as the camera pass, accumulating radiance in the same units before optional diffusion and tonemapping.
  - `camera_path_integrator.c` – Hybrid path tracer split into three stages (orthographic top-down view; every pixel shares the same view direction):
    1. **Direct pass** – Every pixel is converted to world-space through the shared `Camera`, checked for line-of-sight to the light using uniform-grid shadow rays, and (when visible) assigned the unified inverse-square falloff seeded with the light’s disk radius/area. Radiance is written additively via tile grids or the linear buffer so indirect light can layer on top naturally before the higher exposure tonemap kicks in.
    2. **Indirect pass** – For pixels that stayed black after the direct sweep, the integrator fires 48 jittered “feeler” rays per pixel. Each hit point looks up its owning `SurfaceSegment` (with circle hits falling back to their analytic normals), checks that the segment faces the pixel, that the light strikes the sampled spot, and that the segment can see the pixel (all resolved with uniform-grid occlusion tests). Incoming energy is estimated from the irradiance cache if available (otherwise falls back to `ComputeDirectRadiance` at the hit point), then every valid contribution is tempered by the shared inverse-square + exponential decay, a sensor-facing term, and per-sample/per-pixel clamps before being *summed* through the additive accumulator so multiple surfaces can light the same sensor without fighting.
    3. **Tonemap pass** – Converts accumulated energy to 8-bit output using an exponential exposure curve (exposure 0.2, gamma 0.55). When tiles are enabled each tile tonemaps independently before writing into the shared pixel buffer.
  - `camera_path_integrator_disney.c` – Experimental 2.5D MIS path tracer (paused in the UI while the renderer is kept 2D).
- `ray_tracing2.c` – Orchestrates renderer lifecycle, resource ownership, blur post-process, and delegates to the selected integrator.
- `render_helper.c` – SDL drawing helpers for circles, polygons, UI text, and light-strength calculations shared by the renderer and editors.
- `uniform_grid.c` – Builds the reusable grid accelerator backed by per-cell object lists. Bounds expand around the current scene, every object splats its AABB into the grid, and traversal uses a 2D-DDA to test only geometry inside each traversed cell. Every ray (camera feelers, light shadow checks, cache occlusion probes) shares the same `UniformGridTraceRay` so hit filtering and epsilon offsets stay identical between subsystems.
- `material_bsdf.c` – Converts scene objects into compact material descriptors and implements shared Lambert/GGX BSDF evaluation utilities used by both integrators.
- `runtime_material_texture_3d.c` – App-local procedural material texture sampler for native `3D` hits and Material preview. It supports triangle-barycentric UV anchoring plus explicit placed-UV sampling for editor face islands, with rust/fog overlays controlled by a node-ready parameter block: pattern mode, coverage, grain, edge softness, contrast, flow, color depth, surface damage, and seed.
- `runtime_material_payload_3d.c` – Resolves scene-object material presets into the native `3D` shading payload. Runtime hits now carry a per-object local triangle ordinal so generated Material face-group texture overrides can be sampled as cohesive face islands and applied to actual BSDF roughness, reflectivity/specular weight, diffuse response, color, and transparency; color depth and surface damage control how strongly the texture changes color and material response.
- `space_mode_adapter.c` – Central mode adapter seam for world/screen conversion dispatch plus ray/hit setup helpers. Current `SPACE_MODE_3D` path preserves 2D behavior intentionally until later parity slices.
- `ray_tracing_mode_backend.c` – Backend route resolver for runtime mode dispatch. It keeps `2D` as canonical, preserves compat-fallback `3D` for unsupported/incomplete scenes, and now activates bounded native `3D` only when retained seeds can build a runtime scene with triangles plus authored light/camera samples.
- `runtime_camera_3d_rays.c` – Native `3D` camera projector for the runtime lane. It builds a bounded projector from `RuntimeCamera3D`, emits proof-scene camera rays, and projects runtime world points back to screen space so live light markers can use the same camera contract as native shading.
- `runtime_direct_light_3d.c` – Native direct-light shading rule for the runtime lane. It shades first-hit camera rays using runtime light samples and the native 3D visibility helpers, now drawing from a 16-slot hit-seeded stratified finite-radius light population. Area-light shadow checks start with 4 samples and stop there for clearly fully visible or fully blocked windows, escalating to 8 samples only for mixed, penumbra, or partial-transparency cases; samples whose maximum direct contribution is below the radiance epsilon skip shadow visibility entirely so soft shadows track authored `light.radius` without stable five-point bands or paying the full sample cost everywhere.
- `runtime_diffuse_bounce_3d.c` – Bounded native diffuse-bounce tier for the runtime lane. It owns the dedicated `Diffuse Bounce` renderer branch, reuses the shared primary-hit/direct-light seam, and now adds a fixed-sample secondary scene-response pass so shadowed first-hit surfaces can receive limited indirect lift.
- `runtime_emissive_light_set_3d.c` – Cached native `3D` emissive-light candidate set. It scans runtime triangles during scene capability refresh, collects resolved emissive material triangles with area/weight metadata, and stores a compact cumulative distribution so Disney v2 area-light sampling does not scan the full mesh per receiver hit.
- `runtime_emissive_direct_3d.c` – Native emissive-material area-light sampler. It samples visible area contribution from the current receiver hit through the cached emissive-light candidate set, returns RGB contribution plus sampled/contributing diagnostics, and reports candidate count, selected candidates, visibility rays, and any invalid-cache full-scan fallback. When derived scene capabilities prove emission support can be skipped, it takes the no-emissive fast path instead of sampling mesh emitters.
- `runtime_disney_v2_3d.c` – Experimental Disney v2 native `3D` route for the principled material lane. It consumes cached native `3D` payloads through `RuntimePrincipledBSDF3D`, exposes deterministic lobe/direct/emission/transmission diagnostics plus a route-proof diagnostic summary, and is selectable through the distinct `disney_v2` integrator identity without replacing the shipped Disney combiner. The first stochastic slice records a one-bounce path state with sampled diffuse/specular/transmission lobe, support ray, RGB throughput, BSDF/light PDFs, power-heuristic MIS weights, stochastic direct/BSDF radiance channels, secondary ray/hit counters, explicit light-emitter hit state for finite-radius runtime light proofing, and a resolved secondary-hit payload/principled vertex diagnostic that modulates first secondary contribution. Disney v2 now also records per-vertex MIS accounting for light-sampled versus BSDF-sampled contribution branches, finite-light versus emissive-material emitter classification, aggregate branch contributions, branch-removed radiance diagnostics, ordinary emissive-material endpoint contribution from BSDF support rays, primary-vertex emissive-area contribution from cached ordinary emissive-triangle candidates, emissive-area candidate/visibility/fallback diagnostics, and a bounded mirror/glossy reflection channel that reuses native specular reflection tracing for crisp primary reflection while handing reflected geometry recursion to the Disney v2 transport sidecar.
- `runtime_disney_v2_estimator_3d.c` – Focused Disney v2 estimator policy helper. It owns the first power-heuristic MIS weights (`power=2`) and bounded rough-reflection sample-count policy so estimator changes stay out of the host route file.
- `runtime_disney_v2_transport_3d.c` – Focused Disney v2 recursive transport sidecar. It replaces the earlier hand-authored depth-2 light-participation branch with a bounded recursive vertex loop, preserving the legacy depth-2 `recursivePathState` diagnostic while adding fixed-cap per-vertex loop state, local principled material diagnostics, per-vertex diffuse/specular/transmission lobe resampling, local throughput/PDF/MIS updates, contribution RGB, ray/geometry/emitter/contribution counters, and explicit max-depth, roulette, no-hit, emitter, and negligible-throughput termination reasons under `RuntimePathDepthPolicy3D`. It also owns ordinary emissive-material surface endpoint accumulation for stochastic and recursive BSDF paths, primary and recursive emissive-area sample accumulation through the shared emissive-direct helper, plus Disney v2 reflected-geometry recursion from specular/mirror hits with separate reflected recursive vertex diagnostics, reflected recursive area-light merge accounting, and bounded adaptive rough-glossy sample diagnostics.
- `runtime_disney_v2_transmission_3d.c` – Focused Disney v2 transmission adapter. It derives bounded refracted support samples from the existing native dielectric transport helper, roughens primary-visible glass continuation with deterministic native 3D sampling, applies principled transmission tint to RGB throughput, can continue through nested transparent primary-hit surfaces until an opaque receiver or the shared transmission-depth policy stops the path, composites high-transmission primary hits as camera-through receiver radiance with bounded front-surface specular retention instead of leaving the glass front diffuse as the dominant visible signal, shades opaque transmitted receivers through Disney v2 before falling back to direct-light-only endpoint shading, shades/composites transparent intermediate layers front-to-back so complex transparent meshes can show visible interior/backside layer contribution instead of only attenuating the path, splits thin-walled panes from solid dielectric layers so panes keep stable per-surface throughput while solid glass applies bounded distance absorption from `absorptionDistance`, records a bounded solid-transparent interior-return channel, and now separates alpha-only transparent layers from physical dielectric transmission with object-level medium-stack entry/exit/mismatch diagnostics for Disney v2 visual proofing.
- `runtime_native_3d_denoise.c` – Native `3D` final-resolve reconstruction helper. The shipped Disney route keeps the legacy normal/depth bilateral blur, while Disney v2 uses a separate edge-safe policy that only filters stable same-triangle/same-object interiors after temporal accumulation, rejects clean visual edges, preserves transparent and mirror/glossy pixels, skips high-temporal-activity pixels, and reports raw/reconstructed/preserved/skipped diagnostics through render stats.
- `runtime_native_3d_feature_buffer.c` – Native `3D` first-hit feature-buffer builder for reconstruction and temporal policy. It records normal/depth/material risk channels plus triangle/object identity so Disney v2 denoise can distinguish true same-surface interiors from visually separated coplanar edges.
- `runtime_native_3d_adaptive_sampling.c` – Native temporal-sampling policy helper. It marks first-hit opaque emitter pixels as stable for the emission/transparency tier so later temporal subpasses can skip those pixels while continuing to refine non-emissive and transparent surfaces.
- `runtime_native_3d_render.c` – Native live-render handoff for the bounded runtime lane. It builds `RuntimeScene3D` at the active playback `t`, applies live light/camera overrides, then dispatches through the explicit native `3D` integrator seam. `Direct Light`, `Diffuse Bounce`, `Material`, `Emission / Transparency`, and `Disney` are the shipped native dispatch targets today; the dispatch loop traces each primary camera ray once, resolves visible light-emitter hits against that cached surface hit, resolves the first-hit material payload once, and feeds the cached primary hit/transmittance/payload into the selected integrator on top of the shared RGB substrate.
- `runtime_emission_transparency_3d.c` – Native emission/transparency tier. Its cached-primary-hit entrypoint can skip mesh-emission support only for valid derived capabilities with no emissive surfaces and an opaque/non-emissive current payload, and can bypass transparency support only when valid capabilities plus the current payload prove the scene is opaque/non-transmissive; legacy `ShadeHit`/`ShadePixel` paths keep the full support behavior.
- `runtime_ray_3d.c` – Native `3D` ray/hit foundations for the runtime lane. It owns `Ray3D`, `HitInfo3D`, bounded ray-offset behavior, triangle intersection, first-hit traversal across `RuntimeScene3D` triangles, and camera-facing shading-normal orientation so back-side hits on zero-thickness authored planes can still receive lighting.
- `runtime_visibility_3d.c` – Native `3D` visibility helpers for the runtime lane. It casts bounded offset shadow rays from surface hits toward authored runtime light positions, uses derived scene capabilities to take a single-hit opaque/no-volume blocker fast path when transparent or unresolved surfaces and active lighting volumes are absent, and falls back to RGB transmittance for richer material or volume cases.
- `runtime_volume_3d_sampling.c` – Shared native `3D` volume bounds/density sampling helpers. Camera-path absorption and single-scatter marchers both clip through the same attached-volume bounds and sample the same density contract.
- `runtime_volume_3d_scatter.c` – First bounded native `3D` volume inscatter seam. It accumulates isotropic single-scatter contribution from authored direct light along primary camera segments without opening full stochastic volume bounces.
- `runtime_scene_3d.c` – Native runtime `3D` contract scaffold for the first truthful slice. It freezes the `R0` primitive scope (`plane` + `rect_prism`), records ownership boundaries (renderer truth vs compat `SceneObject` vs preview digest), provides runtime scene/light/camera/triangle container types, and derives scene capabilities from resolved geometry/material/volume state while mirroring legacy material flags for compatibility.
- `runtime_environment_3d.c` – Native runtime `3D` environment-light policy. It resolves legacy ambient/top-fill/background brightness settings plus authored `sky` / `warm_sky` / `neutral` presets into explicit ambient surface fill, background miss brightness/color, and summary-ready contribution flags without moving renderer policy into shared core.
- `runtime_scene_3d_samples.c` – Samples canonical authored light and camera state into `RuntimeLight3D` / `RuntimeCamera3D` records at a normalized timeline parameter. This keeps the native render lane independent from preview-only sampling helpers.
- `runtime_scene_3d_builder.c` – Compiles retained bridge primitive seeds into native runtime primitives and deterministic triangle output for the bounded `plane` + `rect_prism` slice, then attaches authored runtime light/camera samples for the same evaluation `t`. Generated plane triangles are marked `twoSided` for zero-thickness wall/floor authoring, while rect prisms and imported/runtime mesh triangles retain single-sided metadata. This is the first render-owned geometry + view/light build path for `R1`.
- `surface_mesh.c` – Tessellates every scene object perimeter into short `SurfaceSegment`s with outward normals plus an index range per object, and now simultaneously emits a per-object `TriangleMesh` derived from the same `SegmentPath`. The triangle view is staged for triangle-accurate visibility while the segment list still powers the indirect heuristic.
- `irradiance_cache.c` – Samples each object perimeter (arc-length parameterised for polygons, angle-based for circles), stores the world-space hit point + normal, then fills eight hemisphere bins. Each bin integrates direct-light energy along jittered rays in that direction (by sampling the prior frame’s energy buffer), but now stops at the first uniform-grid occluder and applies an exponential distance decay before recording the mean/variance plus a one-shot occlusion distance. The camera integrator queries the closest cache entry per hit when estimating indirect energy so specular-ish highlights stay stable even when few feeler rays make it to a surfel.
- `uniform_grid.c` – Builds the reusable grid accelerator backed by per-cell object lists plus the new triangle buckets. Bounds expand around the current scene, every object splats its AABB into the object cells, and every triangle face splats into the triangle cells so ray traversal can test triangle hits first and fall back to analytic circles/polygons when needed. Every ray (camera feelers, light shadow checks, cache occlusion probes) shares the same `UniformGridTraceRay` so hit filtering and epsilon offsets stay identical between subsystems.
- `runtime_native_3d_sampling.c` – Shared native `3D` stratified/blue-noise sampling helpers used by the stochastic transport tiers.
- `runtime_native_3d_progress_hud.c` – Runtime-only native `3D` preview HUD that tracks temporal subpass/tile completion plus elapsed/estimated timing, drawing a small bottom-right progress card on the live renderer without contaminating exported frame buffers.
- `runtime_native_3d_blue_noise.c` – Small deterministic blue-noise support layer that feeds the shared native `3D` sampling seam.
- `runtime_native_3d_temporal_accum.c` – Native `3D` temporal accumulation and resolve helpers. The current lane owns the RGB-aware accumulation substrate plus the bounded EMA/firefly-clamp upgrade shared by tiled and full-frame resolves.
- `forward_light_integrator.c` now exposes user-tunable falloff distance and selectable attenuation modes (none/linear/quadratic) so the SDL menu can keep forward lighting readable across large frame sizes without hard-coded decay.
