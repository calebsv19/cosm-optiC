# optiC Agent Demo Pack

This is the canonical RT-R2 demo pack for fresh local agents. It uses existing
repo fixtures and ignored generated output roots; it does not require the
desktop app, release artifacts, production registry writes, website publishing,
or remote worker jobs.

Path convention: commands below assume the public GitHub repo was cloned into a
directory named `ray_tracing`:

```bash
git clone https://github.com/calebsv19/cosm-optiC ray_tracing
```

If your checkout directory is `cosm-optiC`, replace `make -C ray_tracing ...`
and `ray_tracing/...` paths with `make -C cosm-optiC ...` and
`cosm-optiC/...`.

## Pack Contents

| Role | Input | Command | Expected Output |
| --- | --- | --- | --- |
| Shortest render smoke | `tests/fixtures/agent_render_image_export_request.json` | `make -C ray_tracing test-ray-tracing-render-headless-image-export` | two BMP frames plus `render_summary.json` under `build/agent_runs/ray_tracing/image_export_smoke/` |
| Canonical scene demo | `tests/fixtures/mesh_asset_runtime_spheres/scene_runtime.json` and `tests/fixtures/agent_render_mesh_asset_spheres_request.json` | `make -C ray_tracing test-ray-tracing-render-headless-mesh-asset-spheres` | one direct-light BMP and summary for the default route plus TLAS/flattened route comparison outputs |

| Queue bundle dry run | `tools/export_worker_queue_fixture.py` fixture mode | local export plus queue validate commands below | one `scene-only` queue item under `visual_artifacts/worker_queue_exports/` |

## Canonical Scene

Use the mesh-asset runtime spheres fixture as the first richer demo:

```text
ray_tracing/tests/fixtures/mesh_asset_runtime_spheres/
```

It contains one room-style runtime scene and generated mesh-asset sidecars. The
base scene has three mesh instances:

- `obj_sphere_low` -> `asset_sphere_8x4` with `48` triangles
- `obj_sphere_medium` -> `asset_sphere_16x8` with `224` triangles
- `obj_sphere_high` -> `asset_sphere_32x16` with `960` triangles

The canonical request is:

```text
ray_tracing/tests/fixtures/agent_render_mesh_asset_spheres_request.json
```

It renders one `160 x 96` direct-light frame with `temporal_frames = 1` and
writes to:

```text
ray_tracing/build/agent_runs/ray_tracing/mesh_asset_spheres_mrt4/
```

Expected summary checks:

- `schema_version = ray_tracing_headless_summary_v1`
- `scene_applied = true`
- `route_native_3d = true`
- `prepared_frame = true`
- `rendered_frames = true`
- `frames_rendered = 1`
- `prepared_acceleration.active_trace_route = tlas_blas`
- all three sphere objects report positive primary-hit pixels

## Shortest Render Smoke

Use this when the agent only needs to prove the headless render CLI is usable:

```bash
make -C ray_tracing test-ray-tracing-render-headless-image-export
```

Expected output root:

```text
ray_tracing/build/agent_runs/ray_tracing/image_export_smoke/
```

Expected files:

- `render_summary.json`
- `render_progress.json`
- `frames/frame_0000.bmp`
- `frames/frame_0001.bmp`

Both frames should be nonempty BMP files.

## Material Surface Demo

Use this when the question is material/surface readback rather than full lighting
transport:

```bash
make -C ray_tracing test-ray-tracing-material-preview-headless
```

The material-preview test builds a small authored scene through `line_drawing`,
then runs `ray_tracing_material_preview_headless` on the selected object. The
expected output is a BMP contact sheet plus a JSON summary with variant
readback.

Promoted material preview sets live under:

```text
ray_tracing/docs/material_preview_sets/
```

## Queue Bundle Dry Run

Use this to prove the portable `scene-only` worker bundle shape without live
remote submission. The export command is repo-local. The validation command
uses the broader CodeWork workspace helper `bin/vps_worker_job_queue.py`; skip
that validation step in a standalone public clone unless that helper is present
beside the checkout.

```bash
python3 ray_tracing/tools/export_worker_queue_fixture.py \
  --fixture \
  --output-root ray_tracing/visual_artifacts/worker_queue_exports \
  --item-name ray-tracing-agent-demo-pack
python3 bin/vps_worker_job_queue.py \
  --queue-root ray_tracing/visual_artifacts/worker_queue_exports \
  validate \
  --item-name ray-tracing-agent-demo-pack
```

Expected bundle contents include:

- `bundle/scene_runtime.json`
- `bundle/render_request.json`
- `bundle/request/job.json`
- `bundle/request/payload/...`
- `assets/mesh_assets/*.runtime.json`

This is a local validation lane only. Do not upload, submit, refresh worker
packages, or publish visualizer output as part of RT-R2.
